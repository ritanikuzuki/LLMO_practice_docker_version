LLMO最適化された記事自動生成ツール

システム化の背景・目的
背景
・	生成AIの台頭により、これまでのweb重要視されていたSEOに代わり生成AIからの引用のされやすさを左右するLLMOがより重要になった。
・	AIに引用されるコンテンツ制作のノウハウがなく、社内システムでの支援が必要
目的
・	LLMOに最適化された記事を生成して多くの人の目に触れられるようにする
・	LLMO最適化の知識がなくても最適化された記事を生成できる

解決するべき問題・課題
・	LLMOを上げるための要素を取り入れる
・	どこのプラットフォームに投稿するかによってテーマや記事の内容、書き方が変わる
  ・例としては
  ・Qiita	: エンジニア向けの実務寄り技術記事
  ・Zenn	: コアで数式や論文なものを取り扱う技術記事
  ・Note	: 個人の学びなどの比較的ラフなもの
・	MarkDownを生成して記事のプラットフォームに張り付けて投稿するのが面倒だったため、　このシステム内だけで完結できるようにする
・	実用を見据えてサーバー側にコピーして動かす際にスムーズに動かすためにDockerを使ってコンテナ化する




　
解決策
・	LLMOをgoogle で検索してなにがLLMO最適化に寄与するのかをリサーチした
  ・具体的な数字などで簡潔かつ具体的に書く
  ・構造化・明示的な表現
  ・FAQを設置
  ・E-E-A-T(経験、専門性、権威性、信頼性)を満たす
  ・筆者自身の体験に基づく情報（一次情報）を含める
  ・
  ・参考（https://mieru-ca.com/ai-seo/about_llmo/ 、https://www.ecbeing.net/ai/contents/detail/48）
・	QiitaのアクセストークンAPIを使うことで、このシステムだけで完結することができるようになった



使用方法
1.	生成コンテンツの主題となるメインキーワードを入力
2.	概要や具体的な情報（経験談など）があれば入力
3.	投稿予定のプラットフォームを選ぶ
4.	生成ボタンを押す
5.	プレビュー画面でよければマークダウンをコピーして投稿
6.	qiitaを選ぶ場合はトークンをコピーして投稿可能
