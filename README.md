# LLMO用コンテンツ生成・投稿支援システム (CloudFlow Dynamics)

## 📌 企画背景と着眼点

急成長中のSaaS企業「CloudFlow Dynamics」のマーケティングチームが抱える課題を解決するためのシステムです。
近年、情報の検索においてGoogleなどの従来の検索エンジンから、ChatGPT、Claude、Gemini等の**生成AIによる検索・回答生成**へのシフトが進んでいます。

**【着眼点：SEOからLLMOへのパラダイムシフト】**
従来のSEO（検索エンジン最適化）は人間が読むためのキーワード配置が主でしたが、これからの時代は**LLMO（LLM最適化：AIに引用されやすい構造化）**が必須です。しかし、社内には「AIに好まれる構造」を意識したコンテンツ制作のノウハウがなく、手動での対応には限界がありました。
そこで、**「キーワードを入れるだけで、AIが引用しやすい構造のコンテンツを自動生成し、そのまま公開まで完結できる」**仕組みを企画・実装しました。

## 💡 アプローチと工夫した点（思考のプロセス）

本課題において、単にAPIを叩いて文章を作るだけでなく、実際のマーケティング現場での「使われやすさ」と「成果の出やすさ」に重きを置いて設計しました。

1. **LLMに引用されやすいプロンプトエンジニアリングとE-E-A-Tの強化**
   - 「見出しの階層構造」「明確な定義文（〜とは）」「箇条書き」「表形式」など、AIが情報をパース・抽出・引用しやすい形式をプロンプト内に組み込んでいます。
   - さらに、LLMOにおいて最も重要な**「独自の一次情報の提供（E-E-A-Tの確立）」**をシステムに組み込んでいます。ユーザーに「独自の経験談やデータ」の入力を促し、それをストーリーテリング形式で記事内に自然に織り交ぜるようAIに指示することで、単なる情報まとめではない「唯一無二のコンテンツ」を生成します。
2. **プラットフォーム別の最適化出力**
   - Qiita、Zenn、noteなどのプラットフォームごとに、読者層やAIの学習・参照傾向が異なります。そのため、選んだプラットフォームに合わせて、文体やフォーマット（コードブロックの有無、YAMLフロントマターなど）を自動で切り替えるロジックを実装しました。
3. **「実行（公開）」までを支援するダイレクト投稿機能**
   - マーケティングチームの最大の課題は「記事を作って終わり」になってしまうことです。そこで、作成したコンテンツをワンクリックでプラットフォーム（現在はQiitaのみ対応）に公開できる機能を追加し、**「運用をやり切る」ことまでをシステムで担保**する設計にしました。
4. **実現可能性とスピードを意識した技術選定**
   - 不要に複雑なモダンフロｐｄトエンド（React/Next.jsなど）は使わず、あえてFlaskとHTML/CSS/JS（jQuery等も不使用）というシンプルな構成を選択しました。これにより、プロトタイプの開発速度を最大化し、Dockerコンテナ化による社内への即時デプロイを可能にしています。

## 🚀 機能一覧（使用方法）

1. **要件入力**: 主題となるメインキーワードと、記事の概要（または既存コンテンツ）を入力します。
2. **プラットフォーム選択**: Qiita / Zenn / note / 一般ブログ から投稿先を選択します。
3. **タイトル選定**: AIが読者の課題や関心事を想定したタイトル案を5〜10個生成します。最適なものを1つ選びます。
4. **記事生成**: 選んだタイトルに基づき、LLMOに特化したMarkdown記事が自動生成されます。
5. **公開・出力**: プレビュー画面で内容を確認し、Markdownをコピーするか、**Qiitaアクセストークンを入力して直接記事を公開**することができます。

## 🛠 使用技術と選定理由

- **Backend**: Python 3.11, Flask
  - 選定理由: AI連携（OpenAI SDK）の実装が容易であり、軽量なWebアプリを最速で構築できるため。
- **Frontend**: HTML5, Vanilla CSS, Vanilla JavaScript, Markdown.js
  - 選定理由: ブラウザのみで動作し、追加のビルド環境が不要なため。マーケティングチームの誰もがアクセスしやすいシンプルなUIを構築。
- **AI / API**: OpenAI API (gpt-4o-mini相当), Qiita API (v2)
  - 選定理由: 汎用性の高い文章生成と、日本国内の主要なエンジニア向けプラットフォームへの直接投稿を実現するため。
- **Infrastructure**: Docker, Docker Compose
  - 選定理由: 「手元のPC環境に依存せず、誰もがコマンド一発（`docker-compose up`）でシステムを立ち上げられる」再現性を確保するため。

## ⚙️ ローカルでの起動方法 (デモ準備用)

1. リポジトリをクローンします。
2. プロジェクトのルートディレクトリに `.env` ファイルを作成し、OpenAIのAPIキーを設定します。
   ```env
   OPENAI_API_KEY=sk-your-api-key-here
   ```
3. Docker Composeを使用して起動します。
   ```bash
   docker-compose up --build
   ```
4. ブラウザで `http://localhost:5000` にアクセスします。
